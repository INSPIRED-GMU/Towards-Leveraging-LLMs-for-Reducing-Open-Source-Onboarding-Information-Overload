 Contributing to RxJava 3.x
Want to Add Your Code?
Great! You can contribute through GitHub. Here's how:
1. Make your own copy of the RxJava repository by forking it.
2. Work on your changes and then send them back to us through a pull request. Aim to target the `3.x` branch with your changes.

Writing Your Code
Follow the Style: Make sure your code looks like the rest of RxJava. This helps everyone read and understand it faster.

 Licensing Your Contribution
License Agreement: By sending us your code, you're okay with it being under the Apache 2.0 license, which you can find here: APLv2 License (link).

New Files: If you're adding new files, please start them with a header like this:

This header mentions the license and makes it clear how the file can be used.

That's it! Thanks for considering contributing to RxJava. Your help makes the project better for everyone.
